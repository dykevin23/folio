# Folio 기획 문서

> 개인 자산 관리 & 부동산 트래킹 앱

---

## 목적

PC에서 엑셀로 관리하던 부동산 시세 트래킹 작업을 모바일에서도 어디서든 할 수 있도록 하고,
TODO 리스트와 자산까지 전체적으로 관리하는 개인용 웹 앱.

- 부동산 호가는 시간이 지나면 어디서도 찾을 수 없음 → 직접 매월 수집/기록
- 트래킹한 부동산 데이터의 분석/지표를 어디서든 확인
- 모바일에서 빠른 입력, PC에서 분석/관리

---

## Feature 구성

| Feature | 메뉴 | 상태 |
|---------|------|------|
| TODO 리스트 | 홈(대시보드) 내 포함 | 1차 개발 |
| 부동산 트래킹 | 부동산 | 1차 개발 |
| 자산 관리 | 자산 | 추후 개발 |
| 설정 | 설정 | 각 feature와 함께 |

---

## 플랫폼별 역할

| 플랫폼 | 주요 역할 |
|--------|---------|
| Mobile | 빠른 데이터 입력 + 주요 지표 확인 |
| PC | 데이터 분석, 차트, 상세 관리 |

---

## 네비게이션 구조

### Mobile — 하단 탭 바
```
[ 홈 | 부동산 | 자산 | 설정 ]
```

### PC — GNB (상단)
```
[ Folio로고 ]  홈 | 부동산 | 자산 | 설정
```
- GNB는 메인 메뉴 4개
- 각 페이지 내 로컬 탭으로 세부 화면 전환

---

## 화면 설계

---

### 1. 홈 (대시보드)

#### [Mobile]

카드 형태로 각 항목을 구분하여 표시.

**1-1. TODO 카드**
- 오늘의 TODO 리스트 표시
- 각 항목 상태를 카드에서 바로 업데이트 가능 (별도 페이지/팝업 전환 없이)
- 추가 버튼 → 바텀시트로 할일 추가
- 카드 상단 또는 카드 자체 탭 → TODO 전용 페이지로 전환
  - 날짜 이동하며 해당 날짜의 TODO 확인
  - 추가 / 삭제 기능 포함
- 월간 목표 입력
- 당월 실적 확인

**1-2. 부동산 카드**
- 카드 여러 개 또는 탭 형태로 구성
- 보고 싶은 지표/분석 데이터 표시 (세부 구성은 데이터 입력 후 결정)

**1-3. 자산 카드**
- 자산 항목별 현황과 변동사항
- 주식 종목 리스트 스타일로 표시

#### [PC]

대시보드 비중 (최후 작업):
- 부동산 관련 데이터 70% (차트 중심)
- 자산 20%
- TODO 관련 10%

---

### 2. 부동산

#### [Mobile]

- 핵심: 앞마당(지역구)별 아파트 시세를 빠르게 입력
- 앞마당 기본정보 확인
- 아파트 기본정보 확인

#### [PC]

로컬 탭 구조:

**2-1. 앞마당 (지역구)**
- 앞마당 기본정보
- 입지가치 기준별 평가점수 (정량지표)
- 생활권 우선순위 (선호도)
- 랜드마크 단지 및 가격 순위

**2-2. 단지 목록**
- 앞마당 내 시세 트래킹 중인 아파트 목록

**2-3. 아파트 상세**
- 단지 목록에서 단일 선택 시 진입
- 아파트 기본정보
- 가격정보 및 월별 호가 입력
- 가격 변화 차트

**2-4. 시세 그룹핑**
- 앞마당별 아파트 가격을 생활권별 / 억단위별로 그룹핑
- 한눈에 가격대 분포 확인

**2-5. 5순위 시세표**
- 월 1회 기준
- 앞마당별 가격대별 5순위
- 전체 앞마당 통합 가격대별 5순위
