# Feature: 부동산 트래킹

## 개요

지역별 아파트 단지의 호가/실거래가를 매월 직접 수집하여 입력하고, 가격 변화를 분석하는 기능.
호가는 시간이 지나면 어디서도 찾을 수 없기 때문에 직접 데이터로 관리한다.

---

## 데이터 계층

```
시/구 (regions)
  └── 생활권 (zones)          ← 사용자 임의 정의 구역
       └── 아파트 단지 (complexes)
            └── 평형 타입 (complex_types)  ← 59 / 84
                 └── 월별 가격 (price_histories)
```

---

## 모바일 UX (입력 중심)

- 지역 > 생활권 > 단지 탐색 및 선택
- 단지별 평형 타입 선택 후 이번 달 가격 입력
  - 매매 호가 / 전세 호가
  - 매매 갯수 / 전세 갯수
  - 매매 실거래가 / 전세 실거래가 (nullable)
  - 신고가 (nullable, 갱신 시만)
- 단지 신규 등록 폼

---

## PC UX (분석 중심)

- 구 단위 그룹 보기
- 생활권별 단지 목록
- 단지 상세: 가격 추이 차트 (매매/전세 호가 월별 변화)
- 단지 간 가격 비교
- 전고점 대비 현재가 비율 표시
- 호가 vs 실거래가 갭 시각화

---

## 단지 정보 상세

### 기본 정보
| 항목 | 설명 |
|------|------|
| 아파트명 | |
| 지역 | 시 > 구 > 동 (법정동) |
| 생활권 | 사용자 임의 정의 |
| 연식 | 건축 연도 |
| 세대수 | |
| 구조 | 계단식 / 복도식 / 복합식 |
| 방/화장실 | 방 개수, 화장실 개수 |
| 네이버 부동산 URL | |

### 분석 정보
| 항목 | 설명 |
|------|------|
| 선호도 | 1~5 (5=상, 4=중상, 3=중, 2=중하, 1=하) |
| 입지가치 (location_values) | job / traffic / env / school 중복 선택 |
| 전고점 / 전고점 시기 | 수동 입력, 거의 변경 없음 |
| 전저점 / 전저점 시기 | 수동 입력, 거의 변경 없음 |

### 평형 타입
- 단지 하나에 복수의 타입 존재 (예: 59타입, 84타입)
- 공급면적/전용면적은 참고용으로 저장
- 가격 데이터는 타입 단위로 관리

### 월별 가격 데이터
| 항목 | 설명 |
|------|------|
| record_date | 매월 1일 기준 (YYYY-MM-01) |
| 매매 호가 | |
| 전세 호가 | |
| 매매 갯수 | 현재 매물 수 |
| 전세 갯수 | 현재 매물 수 |
| 매매 실거래가 | nullable |
| 전세 실거래가 | nullable |
| 신고가 | nullable, 갱신 시만 입력 |
